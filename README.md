# vk_sta_spam_may
В качестве выполнения задания были использованы 3 модели: случайный лес, градиентный бустинг и логистическая регрессия. Каждая из моделей показала примерно одинаковый хороший результат. Финальная оценка скора для теста была сделана с помощью логистический регрессии, потому что другие модели не дали сильно лучшего результата, а она сама является простой в плане хранения(памяти) и нужных вычислительных мощностей. К тому же лес и бустинг немного уходили в переобучение.
Предварительно файл test_spam.csv был обработан с помощью Analyse&FeatureEng.ipynb и превращен в prepared_test.csv. На нем и делалось предсказание, а остальные столбцы-признаки были отброшены.
В качестве токенайзера был использован nltk.TweetTokenizer, потому что он хорошо обрабатывает смайлики.
